# 命令行

## 命令格式

```bash
命令 【-选项】 【参数】
```

- 【】符号表示非必选项
- 当有多个选项时，可以写在一起
- 简化选项(一个-)与完整选项(两个--），比如Is -a等于ls --all
- 在一行内运行多个命令，可以使用分号 `;` 进行隔开。例如：

  ```bash
  命令1; 命令2; 命令3
  ```

```bash
ls -a 列出当前目录内容(包括隐藏内容)
ls -l 以长格式列出当前目录内容
ls -al 组合选项
```

## 登录命令login

- Linux是一个真正的多用户操作系统,，可以同时接受多个用户登录，还允许一个用户
  进行多次登录。
- 因为Linux和许多Unix一样，提供了虚拟控制台的访问方式，允许用户在同一时间进行多次登
  录。(alt + F1~F6切换不同的虚拟控制台)
- 每个虚拟控制台可以看作是一个独立的工作站，工作台之间可以切换。

```bash
last (查看最近的登录记录)
last -n 3(查看最近的3条登录记录)
last -3 (等价与上一条)
last tty2 (显示指定控制台登录记录)
last root (显示指定用户登录记录)
exit (退出登录)
top (监视当前的进程，显示PID)
kill PID  关闭进程
```

## 电源命令

###  shutdown

- `shutdown`命令的作用是关闭计算机，使用权限是超级用户

- 主要参数

  ```bash
  -h (关机后关闭电源)
  -r (关机后打开电源(相当于重启))
  -k：并不真正关机，只是送警告信号给每位登录者
  -time:设定关机前的时间
  ```

- `shutdown`命令可以安全地将系统关机，使用直接断掉电源的方式来关闭Linux系统十
  分危险。

- Linux后台运行着许多进程，所以强制关机可能会导致进程的数据丢失，使系统处于不稳定的状态，甚至在有的系统中会损坏硬件设备。

- 在系统关机前使用shutdown命令，系统管理员会通知所有登录的用户系统将要关闭，
  并且login指令会被冻结，即新的用户不能再登录。

###  halt

- halt命令的作用是关闭系统使用权限是超级用户

- 主要参数

  ```bash
  -n 防止sync系统调用
  ```


### reboot

- reboot命令的作用是重新启动计算机，使用权限是系统管理者

- 主要参数

  ```bash
  -n  保存数据后再重新启动系统
  -i  关闭网络设置之后再重新启动系统
  ```

###  文件命令

1. `pwd`
   - 用于<u>打印当前的工作目录</u>
   - 两个选项
     1. `-L`：目录连接链接时，输出连接目录
     2. `-P`：输出物理路径

2. `cd`
   - 用于<u>改变当前工作目录</u>。
   - 语法 `cd [dir]`
     1. `cd ..`进入 (退到)上一层目录两个点代表父目录
     2. `cd~`进入家目录
     3. `cd -`进入前一个目录，适用于在两个目录之间快速切换

3. `ls`
   - Linux 命令中使用频率最高者之一，用于<u>列出目录的内容</u>，或者文件的信息该命令的输出结果默认按照文件名排序，如果不指定目标，则列出当前目录的内容。
   - 语法 `ls [OPTION] [FILE]`
     1. `-a`显示所有文件及目录，包括隐藏的文件和目录。
     2. `-l`除文件名称外，亦将文件型态、权限、拥有者、文件大小等资讯详细列出。
     
        ![image-20240928205350514](C:\Users\30703\AppData\Roaming\Typora\typora-user-images\image-20240928205350514.png)
     3. `-t` 将文件依建立时间之先后次序列出。
     4. `-R` 若目录下有文件，则以下之文件亦皆依序列出。
     5. `-S`按文件大小排序列出
   
4. `mkdir`
   - `mkdir`命令用于<u>创建目录</u>(文件夹)
   - 语法： `mkdir [OPTION]... DIRECTORY...`
     - `-p`如果没有父目录则自动创建

5. `cp`
   - 用于<u>复制文件</u>或者目录，可以一次复制单个文件，也可以一次复制多个文件
     <u>(cp 命令属于高危命令，使用不慎就会有丢失数据的危险)</u>
   - 语法：`cp [OPTION]... SOURCE... DIRECTORY`
     1. `-a`：此选项通常在复制目录时使用，它保留链接、文件属性，并复制目录下的所有内容。
     2. `-p`：除复制文件的内容外，还把修改时间和访问权限也复制到新文件中。
     3. `-r`：若给出的源文件是一个目录文件，此时将复制该目录下所有的子目录和文件。
     4. `-l`：不复制文件，只是生成链接文件

6. `mv`
   - 用于<u>移动文件</u>或者目录<u>（高危命令，使用不慎就会有丢失数据的危险）</u>
   - 如果原文件和目标文件在同一个父目录里面，则mv命令的效果就相当于给文件改名
   - 语法：`mv[选项]源文件或目录 目标文件或目录`
     1. `-b`：若需覆盖文件，则覆盖前先行备份。
     2. `-f`：force强制的意思，如果目标文件已经存在，不会询问而直接覆盖
     3. `-i`：若目标文件（destination）已经存在时，就会询问是否覆盖
     4. `-u`：若目标文件已经存在，且 source 比较新，才会更新(update)。

7. `rm`
   - 用于<u>删除文件</u>或者目录
   - rm命令属于高危命令，没有一个工具能够100%恢复rm命令删除的文件(没有回收站)
   - 语法:`rm [OPTION] file_or_dir`
     1. `-f`, `--force`忽略不存在的文件，从不给出提示。
     2. `-i` `--interactive` 进行交互式删除。
     3. `-r`,` -R`,` --recursive`指示rm将参数中列出的全部目录和子目录均递归地删除
     4. `-v`, --verbose详细显示进行的步骤。

8. `cat`
   
   - 用于<u>读取文件的全部内容</u>，或者将几个文件合并为一个文件。
   - 语法：`cat [OPTION] [FILE]`
     1. `-A`, `--show-all`等价于-vET。
     2. `-b`,` --number-nonblank`对非空输出行编号
     3. `-E,` `--show-ends`在每行结束处显示$。
     4. `-n`,` --number`对输出的所有行编号，由1开始对所有输出的行数编号。
   
9. `head`
   - 用来显示文件的开头内容，默认head命令打印其相应文件的开头10行。
   - 语法：`head [OPTION] [FILE]`
     1. `-q`隐藏文件名(默认隐藏)
     2. `-V`显示文件名
     3. `-c<字节>`显示字节数

10. `tail`
    - 用于<u>读取文件的尾部</u>
    - 语法：`tail [OPTION]... [FILE]`
      1. `-f` 循环读取。
      2. `-q` 不显示处理信息。
      3. `-v`显示详细的处理信息。
      4. `-c<数目>`显示的字节数。
      5. `-n<行数>`显示行数

11. `more`
    - more会以<u>一页一页的显示</u>方便使用者逐页阅读，
    - 而最基本的指令就是按空白键（space）就往下一页显示，按b键就会往回（back）一页显示，而且还有搜寻字串的功能。
    - more命令从前向后读取文件，因此在启动时就加载整个文件。
    - 语法：`more [OPTION]... [FILE]..`
      1. `+n`从第n行开始显示。
      2. `-n`定义屏幕大小为n行。
      3. `+/pattern`在文件显示前搜寻该字串（pattern），然后从该字串前两行之后开始显示。
      4. `-C`从顶部清屏，然后显示。
    - 交互操作
      1. `Enter`向下n行,需要定义。默认为1行
      2. `Ctrl+F`向下滚动一屏
      3. 空格键向下滚动一屏
      4. `Ctrl+B`返回上一屏
      5. 输出当前行的行号
      6. 调用vi编辑器
      7. !命令调用shell，并执行命令
      8. q 退出 more。

12. `less`
    
    - less命令<u>读取内容，分屏显示</u>，
    - 使用less可以随意浏览文件，而more仅能向前移动，却不能向后移动，
    - 而且less在查看之前不会加载整个文件
    - 语法：`less [OPTION]... [FILE]..`
    - 交互操作
      1. /字符串：向下搜索“字符串”的功能。
      2. ?字符串：向上搜索“字符串”的功能
      3. Q 退出less 命令。
      4. 空格键滚动一页
      5. 回车键滚动一行。
    
13. `find`
    - 用来在指定目录下<u>查找文件</u>
    - 可以指定一些匹配条件，如按文件名、文件类型、用户甚至是时间戳查找文件
    - 常用语法：`find [path...] [expression]`
      1. `-name`按照文件名查找文件
      2. `-perm `按照文件权限来查找文件
      3. `-user`按照文件属主来查找文件
      4. `-mtime -n +n `按照文件的更改时间来查找文件

14. `which`
    - 在PATH所指定的目录中查找<u>可执行文件</u>
    - 使用which命令，就可以看到某个系统命令是否存在，以及执行的到底是哪一个位置
      的命令。
    - which命令用来确定外部命令的绝对路径
    - 语法: `which [options] programname [...]`

15. `gzip`

    - 在Linux系统中经常使用的一个对文件进行压缩和解压缩的命令
    - 不仅可以用来压缩大的、较少使用的文件以节省磁盘空间。
    - 语法：`gzip [参数] [文件或者目录]`
      1. `-d`或`--decompress`或`----uncompress`解开压缩文件。
      2. `-f`或`-force`强行压缩文件，不理会文件名是否存在以及该文件是否为符号连接。
      3. `-l`或`--list`列出压缩文件的相关信息。
      4. `-r`或`--recursive`递归处理，将指定目录下的所有文件及子目录一并处理。
      5. `-v`或`--verbose`显示指令执行过程。

16. `tar`

    - 可用于打包文件，把多个文件打到一个包中，方便数据的移动。
    - 语法：`tar [OPTION...] [FILE]`
    - `-c`建立新的压缩文件
    - `-×`从压缩的文件中提取文件
    - `-t`显示压缩文件的内容。
    - `-z`支持gzip解压文件
    - `-j`支持bzip2解压文件。
    - `-v`显示操作过程。

17. `ln`

    - In命令用于创建链接文件

    - Linux中有软链接和硬链接两种

      ![image-20240926122451696](C:\Users\30703\AppData\Roaming\Typora\typora-user-images\image-20240926122451696.png)

    - `ln`命令在不带参数的情况下，默认创建的是硬链接。

    - 语法: `ln [-f | -n] [-s] SourceFile [TargetFile]`

      1. `-b` 删除，覆盖以前建立的链接。
      2. `-d`允许超级用户制作目录的硬链接
      3. `-f `强制执行。
      4. `-i`交互模式，文件存在则提示用户是否覆盖
      5. `-n`把符号链接视为一般目录
      6. `-s`车软链接(符号链接)

# vim编辑器

## 打开文件  

​	`vim filename`

1. 如果filename文件存在，则会打开文件并显示文件内容；
2. 如果filename文件不存在，vim会在下面提示[New File]，并且会在第一次保存时创建该文件。

## 移动光标

1. 上下左右键或k、j、h、I键上下左右移动光标
2. `0`移动到行首
3. `g0`移到光标所在屏幕行行首
4. `:n`移动到第n行
5. `gg`到文件头部,
6. `G`到文件尾部。

## 数据操作

1. `yy or Y:`复制整行文本。
2. `y[n]w`:复制—(n)个词
3. `d[n]w`:删除(剪切)1(n)个单词
4. `[n] dd`:删除(剪切)1(n)行
5. `p`
   - 面向行数据：放置数据在当前行的下面(第一次按)
   - 面向行数据：放置数据在当前行的上面(第二次按)
   - 面向字符数据：p放置数据在光标的后面
   - 面向字符数据：P放置数据在光标前

## 行号 

  - 显示行号 `set nu`

  - 取消显示行号 `set nonu`

## 查找

  - `:word` 

    在光标之后查找一个字符串word，按n向后继续搜索，shift+n向上搜索。


  - `:?word`

    在光标之前查找一个字符串word，按n向后继续搜索

## 替换

  - `:1,5s/word1/word2/g`

    将文档中1-5行的word1替换为word2，不加g则只替换每行的第一个word1.


  - `%s/word1/word2/gi`

    将文档所有的word1替换为word2，不区分大小写。

## 修改文件

  - 使用vimfilename打开文件后，进入的是普通模式。

  - 当想要修改文件时，可以按`i`键进入插入模式

    按ecs 可以退出插入模式，回到普通模式

## 撤销或重做

  1. `u`撤销最近的改变
  2. `U`撤销当前行自从光标定位在上面开始的所有改变
  3. `Ctrl+r`重做最后一次‘撤销'改变

## 保存文件并退出

  1. `:W`保存
  2. `:q`退出
  3. `:wq`保存并退出
  4. `q!`强制退出
  5. `:wq!`强制保存并退出

## 查看文件

1. `cat`
   - 显示文件内容， `cat filename`
   - 编辑一个文件`cat > filename`
   - 将几个文件合并为一个文件`cat file1 file2 > file3`(会覆盖原文件内容)
   - 第2，3个命令使用`>>`两个大于号可以不覆盖原文件,把新增的内容放在原文件末尾
   - 选项
     1. `-n`:从1开始对所有行编号并显示在每行开头
     2. `-b`：从1开始对非空行编号并显示在每行开头
     3. `-s`：当有多个空行在一起时只输出一个空行
     4. `-E`：在每行结尾增加S
     5. `--help`：显示帮助信息


  2. `more`
     - more可以一次查看文件或者标准输入的一页，与cat不同的是more可以按页来查看文
       件的内容，还支持直接跳转行等功能
     - 命令格式：`more [options] <file>...`
     - 选项
       1. `+n`:从第n行开始显示
       2. `-n`：定义屏幕大小为n行
       3. `-c`:从顶部清屏，然后显示
       4. `-s`：把连续的多个空行显示为一行
     - 交互操作
       1. `Enter`:默认向下滚动1行
       2. `Ctrl+F`:向下滚动一屏
       3. `空格键`：向下滚动一屏
       4. `Ctrl+B`：向上滚动一屏
       5. `b`：向上滚动一屏
       6. `=`：输出当前行号
       7. `:f`：车输出文件名和当前行号
       8. `:q`：退出more


  3. `less`		
     - less 可以一次查看文件或者标准输入的一页，less 的用法比起more 更加的有弹性。
     - 可以用光标上下移动来查看文件内容
     - 命令格式：`less [option] 文件`
     - 选项
       1. `-f`:强制打开特殊文件，例如外围设备代号、目录和二进制文件
       2. `-g`:只标志最后搜索到的关键字
       3. `-i`:忽略搜索时的大小写
       4. `-N`:显示每行的行号
       5. `-s`:当有多个空行在一起时只输出一个空行
       6. `-o<文件名>`：将less输出的内容保存到指定文件

     - 交互操作
       1. `b`：向上翻一页
       2. `d`：向下翻半页
       3. `h`：显示帮助界面
       4. `q`：退出less
       5. `u`：向上翻半页
       6. `y`：向上翻一行
       7. `空格键`：向下翻一页
       8. `Enter`:向下翻一行
       9. `上下键`：向上/下翻一行




## 文件摘选

  - `head`
    - head用来显示文件的开头至标准输出中，默认head命令可以显示文件的前10行
    - 命令格式：`head[option]...[文件]..`
    - 选项
      1. `-q`：输出时隐藏文件名，head默认不显示文件名
      2. `-v`：输出时显示文件名
      3. `-c num`:显示前num个字节
      4. `-n num`:显示前num行


  - `tail`
    - 用来显示文件的末尾至标准输出中，默认tail命令可以显示文件的后10行
    - 命令格式：`tailI [option]... [file]..`
    - 选项
      1. 循环读取，对于日志文件的监控非常有用
      2. `-q`：不显示文件名，tail默认不显示文件名
      3. `-v`显示文件名
      4. `-c num`:显示文件最后num个字节
      5. `-n num`: 显示文件最后num行
      6. `-s`：与`-f`结合使用，表示每次反复的时间间隔休眠s秒


  - `cut`
    - 用于显示文件或者标准输入的特定列
    - 命令格式：: `cut [option]..[文件]`
    - 选项
      1. `-b`：仅显示行中指定直接范围的内容(byte)
      2. `-c`[范围：仅显示行中指定范围的字符(character)
      3. `-d`：指定字段的分隔符，默认的字段分隔符为"TAB"
      4. `-f[范围]`：显示指定第num个字段的内容，可以用逗号隔开显示多个字段

# 用户与权限管理

- 用户是能够获取系统资源的权限的集合；
- 每个用户都会分配一个特有的id号-uid

##  UID

- 超级用户：也称为root用户，它的uID为o，超级用户拥有系统的完全控制权，可以进行修改、删除文件等操作,也可以运行各种命令，所以在使用root用户时要十分谨慎；
- 普通用户:也称为一般用户，它的UID为1000-60000之间，普通用户可以对自己目录下的文件进行访问和修改，也可以对经过授权的文件进行访问;
- 虚拟用户：也称为系统用户，它的UID为1-999之间，虚拟用户最大的特点是不提供密码登录系统，它们的存在主要是为了方便系统的管理。

### 区分用户类别

- 查看UID命令：`id [option] [user_name]`
- 相关参数：
  - `-u,-user`只输出有效UID
  - `-n,-name`对于`-ugG`输出名字而不是数值
  - `-r，-real `对于`-ugG`输出真实ID而不是有效ID
- UID为0时，标识的是超级用户（即root用户），UID为1000-60000之间，标识的是普通用户，
  UID为1-999之间，标识的是虚拟用户（即系统用户）。

# 文件权限

##  概述

- 权限是操作系统用来限制对资源访问的一种机制，分为读、写、执行。
- 在Linux系统中，不同的用户所处的地位也不尽相同，不同地位的用户拥有不同的权
  限等级，为了保证系统的安全性，Linux系统针对不同用户的权限制定了不同的规则
- 在Linux系统中，每个文件或目录都具有特定的访问权限、所属用户及所属组，通过
  这些规则可以限制什么用户、什么组可以对特定的文件执行什么样的操作。

# DNF

##  yum

- yum是基于rpm包管理
- 可以从指定的服务器自动下载RPM服务器并进行安装，
- 作为软件仓库对软件进行管理
- 优点
  - 解决了软件包间的依赖关系
  - 提升了效率
- 缺点
  - 性能差，内存占用过多
  - 依赖解析速度变慢
  - 过度依赖yum源文件
- DNF应运而生

## 概述

- 用来管理RPM软件包
- 查询软件包的信息，从指定软件库去获取所需软件包，并通过自动处理依赖关系来实现安装、卸卸载以及更新。
- 与YUM完全兼容，提供了YUM兼容的命令行以及为扩展和插件提供的API
- 使用需要root权限

## DNF配置文件

1. `/etc/dnf/dnf.conf`

   - `cachedir`：缓存目录，该目录用于存储RPM包和数据库文件;
   - `best`：升级包时，总是尝试安装其最高版本，如果最高版本无法安装，则提示无法安装的原因并停止安装。默认值为True;
   - `installonly_limit`：设置可以同时安装“installonlypkgs”指令列出包的数量。默认值为3，不建议降低此值;
   - `clean_requirements_on_remove`：删除在dnf remove期间不再使用的依赖项，如果软件包是通过DNF安装的，而不是通过显式用户请求安装的，则只能通过clean_requirements_on_remove删除软件包，即它是作为依赖项引入
     的。默认值为True。

2. 安装软件源位置

   - 通过vim命令，可以在文件中添加一个或多个“repository" : `vim /etc/dnf/dnf.conf`
   - `name=repository_name`：软件仓库（repository）描述的字符串
   - `baseurl=repository_url`：软件仓库（repository）的地址。

3. 创建本地软件源仓库(root权限)

   - `dnf install createrepo`安装`createrepo`

   - 将需要的软件包放置在目录下，比如`/mnt/local_repo/`;

   - 执行命令创建软件源

     `createrepo --database /mnt/local_repo`

4. 添加软件源(root权限)

   - ` dnf config-manager --add-repo repository_url `
   - 在`/etc/yum.repos.d/`目录下生成对应的repo文件

5. 启用和禁用软件源(root权限)

   - `dnf config-manager --set-enable repository`启用
   - `dnf config-manager --set-disable repository`禁用

## systemd

- systemd是与SysV和LSB初始化脚本兼容的系统和服务管理器，可以提供基于守护进程的按需启动策略。
- systemd服务支持快照和系统状态恢复，维护挂载和自挂载点，使得各服务之间基于从属关系实现更为精细的逻辑控制，具有更高的并行性能。
- `d`指daemon(n. (希腊神话中)半人半神的精灵)，守护进程

###  systemd unit

- systemd服务的开启和监督系统是基于unit的概念，unit由一个与配置文件对应的名称
  和类型组成(<u>单元：让系统知道如何进行操作和管理的资源</u>)
- unit类型
- `Service unit`：系统服务
- `Target unit`: —组systemd units
- `Automountunit`：文件系统挂载点
- `Device unit`:内核识别的设备文件
- `Mount unit`：文件系统挂载点
- `Pathunit`：在一个文件系统中的文件或目录
- `Scope unit`:夕外部创建的进程
- `Snapshot unit`： systemd manager的保存状态

### 特性

- 更快的启动速度
- 提供按需启动能力
- 采用cgroup特性跟踪和管理进程的生命周期
- 启动挂载点和自动挂载的管理
- 实现事务性依赖关系管理
- 与SysV初始化脚本兼容
- 能够对系统进行快照和恢复

### 命令

- 显示服务`systemctl list-units --type service`
- 显示服务状态`systemctl status name.service`
  - `Loaded`：说明服务是否被加载，同时显示对应的绝对路径是否启用
  - `Active`：说明服务是否正在运行，并显示时间节点
  - `MainPID`：相应系统服务PID值
  - `CGroup`:↑相关控制组的其他信息
- 运行服务`systemctl start name.service`
- 关闭服务`systemctl stop name.service`
- 重启服务`systemctl restart name.service`
- 启用服务`systemctl enable name.service`
- 禁用服务`systemctl disable name.service`

## 备份

在数据库和系统数据管理中，备份可以分为**基础备份（物理备份）\**和\**逻辑备份**两大类，主要区别在于备份的数据形式和适用场景：

### 1. 基础备份（物理备份）

- **定义**：基础备份是直接对数据库或系统的物理文件（如数据文件、日志文件等）进行备份。通常备份的是数据文件在存储设备上的物理布局。

- **操作方式**：通过复制数据文件、日志文件等来进行备份，例如使用操作系统的文件复制命令、数据库提供的物理备份工具。

- **适用场景**：数据量大、对恢复速度要求高的数据库（如生产数据库）备份。

- 命令`PITR`

  `PITR`: 全称是Point-In-Time-Recover （时间点恢复），是PG从8.0版本开始引入的一个特性，该特性可以使用**基础备份**和连续归档日志将数据库集群恢复到任意时间点。

### 2. 逻辑备份

- **定义**：逻辑备份是通过导出数据库的结构（如表结构、视图、存储过程等）和内容（即记录）生成的备份文件。备份文件中记录了数据的逻辑信息，而不是存储设备上的物理信息。
- **操作方式**：通常使用数据库自带的逻辑备份工具（如 `mysqldump`、`pg_dump`）。
- **适用场景**：适合数据量较小、需要跨平台迁移的数据库，或用于测试和开发环境。

### 总结对比

- **基础备份**：直接复制物理文件，速度快，适合大数据量，依赖物理架构。
- **逻辑备份**：导出逻辑数据结构和内容，便于迁移和选择性备份，速度较慢。

## 磁盘挂载

- Linux的宗旨是一切皆文件，要想使用磁盘，必须先建立一个联系，这个联系就是一个目录，建立联系的过程我们叫做挂载；
- 当我们访问sdb2底下的这个目录的时候，实际上我们访问的才是sdb2这个设备文件。
- 这个目录相当于一个访问sdb2的入口，可以理解为一个接口，有了这个接口才可以访问这个磁盘。

> #### **挂载**：指的就是将设备文件中的顶级目录连接到 Linux 根目录下的某一目录（最好是空目录），访问此目录就等同于访问设备文件。

### 注意：

- 并不是根目录下任何一个目录都可以作为挂载点
- 挂载操作会使得原有目录中文件被隐藏
- 因此根目录以及系统原有目录都不要作为挂载点，会造成系统异常甚至崩溃
- 挂载点最好是新建的空目录

> **总之，Linux 系统使用任何硬件设备，都必须将设备文件与已有目录文件进行挂载**

# 网络管理

##  OSI 协议栈

- OSI 是Open System Interconnection 的缩写，意为开放式系统互联。
- 把网络通信的工作分为7层，分别是物理层、数据链路层、网络层、传输层、会话层、表示层和应用 层。
- OSI下3层的主要任务是数据通信，
- 第4层是整个网络体系结构中最关键的部分；
- 上3层的任务是数据处理，完成资源子网的功能
- 每一层都提供一个特殊的网络功能。

1. **物理层**(Physical Layer) 

   - 传输数据格式：比特(bit)流； 
   - 功能：
     1. 建立、维护和取消物理连接；
     2. **<u>利用传输介质为数据链路层提供物理连接，实现比特流的透明传输。</u>**
     3. 实现相邻计算 机节点之间比特流的透明传送，尽可能屏蔽掉具体传输介质和物理设备的差异
     4. 使其上面的数据链路层不必考虑 网络的具体传输介质是什么
   - 典型设备：光纤、同轴电缆、双绞线、网卡、中继器、集线器等；
2. **数据链路层**(Data Link Layer)
   - 传输数据格式： 将比特信息封装成数据帧(Frame)；
   - 功能：
     1. 在物理层上建立、撤销、标识逻辑链接和链路复用 以及差错校验等功能。
     2. 通过使用接收系统的硬件地址或物理地址来寻址； 
     3. 负责建立和管理节点间的链路；
     4. 通过各种控制协议，将有差错的物理信道变为无差错的、能可靠传输数据帧的数据链路。
   - 典型设备：网桥、交换机；
   - 分类：
     1. 介质访问控制（MAC）
        - 解决共享型网络中多用户对信道竞争的问题，完成网络介质的访问控制；
     2. 逻辑链路控制（LLC）
        - 建立和维护网络连接，执行差错校验、流量控制和链路控制。 
3. 网络层(Network Layer)
   - 传输数据格式：分割和重新组合数据包(Packet)；
   - 典型设备：网关、路由器；
   - 功能：
     1. 基于网络层地址（IP地址）进行不同网络系统间的路径选择
     2. 是OSI参考模型中最复杂的一层。
     3. 通过路由选择算法，为报文或分组通过通信子网选择最适当的路径
     4. 数据链路层的数据在这一层被转换为数据包
   - 实现
     1. 寻址：在不同子网之间通信时，为 了识别和找到网络中的设备，每一子网中的设备都会被分配一个唯一的地址（IP地址）。
     2. 交换：规定不同的信息交换方式。如线路交换技术和存储转发技术，后者又包括报文交换技术和分组交换技术。
     3. 路由算法：当源节点和目的节点之间存在多条路径时，根据路由算法，通过网络为数据分组选择最佳路径，并将信息从最合适的路径由发送端传送到接收端。 
     4. 连接服务：控制的是从<u>源节点</u>到<u>目的</u>节点间的流量。其目的在于<u>防止阻塞</u>，并进行差错检测。
4. 传输层(Transport Layer)

   - 传输数据格式：数据组织成数据段(Segment)；
   - 典型设备：终端设备(PC、手机、平板等)；
   - 功能
      1. 是通信子网和资源子网的接口和桥梁，起到承上启下的作用；
      2. 向用户提供可靠的端到端的差错和流量控制，保证报文的正确传输。
      3. 向高层屏蔽下层数据通信的细节， 即向用户透明地传送报文。

   - 协议
      - TCP/IP中的TCP协议、
      - Novell网络中的SPX协议
      - 微软的 NetBIOS/NetBEUI 协议。
5. 会话层(Session Layer)

   - 传输数据格式：数据DTPU；
   - 典型设备：终端设备(PC、手机、平板等)；
   - 说明
      1. 是用户应用程序和网络之间的接口
      2. 向两个实体的表示层提供建立和使用连接的方法。
      3. 组织和协调两个会话进程之间的通信，并对数据交换进行管理。
      4. 用户可以按照半双工、单工和全双工的方式建立 会话。
      5. 用户必须提供他们想要连接的远程地址(域名)
   - 功能：
      1. 会话管理：允许用户在两个实体设备之间建立、维持和终止会话，并支持它们之间的数据交换。
      2. 会话流量控制：提供会话流量控制和交叉会话功能。 
      3. 寻址：使用远程地址建立会话连接。 
      4. 出错控制：从逻辑上讲会话层主要负责数据交换的建立、保持和终止，但实际的工作却是接收来自传输层的数 据，并负责纠正错误。
6. 表示层(Presentation Layer)

   - 传输数据格式：数据PTPU；
   - 典型设备：终端设备(PC、手机、平板等)； 
   - 说明
      1. 对来自应用层的命令和数据进行解释，对各种语法赋予相应的含义，并按照一定的格式传送给会话层。
      2. 处理用户信息的表示问题，如编码、数据 格式转换和加密解密等
   - 功能
      1. 数据格式处理：协商和建立数据交换的格式，解决各应用程序之间在数据格式表示上的差异。
      2. 数据的编码：处理字符集和数字的转换。
      3. 压缩和解压缩：为了减少数据的传输量，这一层还负责数据的压缩与恢复。
      4. 数据的加密和解密：可以提高网络的安全性。 
7. 应用层(Application Layer)

   - 传输数据格式：数据ATPU；

   - 典型设备：终端设备(PC、手机、平板等)；
   - 说明
     1. 计算机用户，以及各种应用程序和网络之 间的接口，其功能是直接向用户提供服务，完成用户希望在网络上完成的各种工作。
     2. 负责完成网络中应用程序与网络操作系统之间的联系，建立与结束使用者之间的联系，并完成网络用户提出 的各种网络服务及应用所需的监督、管理和服务等各种协议。
     3. 负责协调各个应用程序间的工作。
   - 提供的服务和协议：文件服务、目录服务、文件传输服务（FTP）、远程登录服务 （Telnet）、电子邮件服务（E-mail）、打印服务、安全服务、网络管理服务、数据库服务等。
   - 功能：
     1. 用户接口：应用层是用户与网络，以及应用程序与网络间的直接接口，使得用户能够与网络进行交互式联系。
     2. 实现各种服务：该层具有的各种应用程序可以完成和实现用户请求的各种服务。

# GaussDB

GaussDB 是华为推出的一款分布式数据库，采用了多层架构，旨在提供高可用、高性能和高扩展性。它的组件主要包括 **DN (Data Node)**、**CN (Coordinator Node)**、**GTM (Global Transaction Manager)** 和 **OM (Operation Manager)** 等，每个组件有着不同的职责。

以下是这些组件的详细介绍：

| 组件                                 | 角色           | 主要功能                                               |
| :----------------------------------- | :------------- | :----------------------------------------------------- |
| **DN (Data Node)**                   | 数据存储节点   | 存储数据，执行数据操作（增、删、改、查）               |
| **CN (Coordinator Node)**            | 协调节点       | 接收客户端请求，解析并调度查询，聚合结果               |
| **GTM (Global Transaction Manager)** | 全局事务管理器 | 负责全局事务的一致性管理，确保分布式事务的正确性       |
| **OM (Operation Manager)**           | 运维管理器     | 数据库的管理和监控，提供运维、故障恢复和自动化管理功能 |



# gs_dump

gs_dump是openGauss用于导出数据库相关信息的工具，用户可以自定义导出一个数据库或其中的对象（模式、表、视图等），回收站对象除外。支持导出的数据库可以是默认数据库postgres，也可以是自定义数据库。

- gs_dump工具由安装openGauss数据库的操作系统用户执行。
- gs_dump工具在进行数据导出时，其他用户可以访问openGauss数据库（读或写）。
- gs_dump工具支持导出完整一致的数据。例如，T1时刻启动gs_dump导出A数据库，那么导出数据结果将会是T1时刻A数据库的数据状态，T1时刻之后对A数据库的修改不会被导出。
- gs_dump工具在进行数据导出时生成的列不会被转储。
- gs_dump支持导出兼容v1版本数据库的文本格式文件。
- gs_dump支持将数据库信息导出至纯文本格式的SQL脚本文件或其他归档文件中。

# SQL 语言

1. **数据查询语言（DQL）**、
   - 从数据库中检索（查询）数据。其主要功能是从数据库中获取所需的记录或信息。最常用的语句是 `SELECT`，它可以检索指定条件的数据。
   - `SELECT`：查询数据。

2. **数据操控语言（DML）**、

   - `INSERT`：插入数据。
   - `UPDATE`：更新数据。
   - `DELETE`：删除数据。
   - `SELECT`：查询数据（尽管也属于 DQL，但在一些上下文中，`SELECT` 被视为 DML 中的一部分，因为它操作数据）。

3. **数据定义语言（DDL）**

   > 用于定义或修改数据库中的对象。如：表、索引、视图、数据库、序列、用户、角色、表空间、存储过程等。

   1. `CREATE`：创建数据库对象（如表、视图、索引等）。
   2. `DROP`：删除数据库对象。
   3. `ALTER`：修改数据库对象的结构。
   4. `TRUNCATE`：删除表中的所有数据，但保留表结构。

4. **数据控制语言（DCL）**。

   用来设置或更改数据库事务、授权操作（用户或角色授权，权限回收，创建角色，删除角色等）、锁表（支持SHARE和EXCLUSIVE两种锁表模式）、停机等。

   1. `GRANT`：授予用户权限。
   2. `REVOKE`：撤销用户权限。

# 用户和角色

1. 定义和目的：

   - 数据库用户：代表一个具体的数据库账户，通常与操作系统用户或网络用户相对应。用户可以登录到数据库并执行操作
   - 数据库角色：是一个预定义的权限集合，可以分配给用户。角色用于简化权限管理，通过将权限分配给角色，然后再将角色分配给用户，而不是直接将权限分配给每个用户。
2. **权限管理**：
   - **数据库用户**：权限是直接分配给用户的，每个用户可以有自己的一组权限。
   - **数据库角色**：权限是分配给角色的，然后角色可以分配给多个用户。这样可以减少重复的权限分配工作，提高管理效率。
3. **灵活性**：
   - **数据库用户**：每个用户可以有独特的权限集合，适合需要精细控制权限的场景。
   - **数据库角色**：通过角色可以快速为一组用户分配相同的权限集合，适合权限需求相似的用户群体。
4. **安全性**：
   - **数据库用户**：用户的权限和操作可以被详细追踪，因为每个操作都是以用户身份执行的。
   - **数据库角色**：通过角色分配权限可以减少直接分配给用户的权限，有助于减少安全风险。

### `cron`

#### 命令`crontab`

cron 服务提供crontab命令来设定cron服务的，以下是这个命令的一些参数与说明： 

- `crontab -u` 设定某个用户的cron服务，一般root用户在执行这个命令的时候需要此参数  
- `crontab -l` 列出用户cron服务的内容  
- `crontab -r` 删除某个用户的cron服务  
- `crontab -e` 编辑某个用户的cron服务(按 X 删除光标所选字符，按 I 在所选位置插入，dd 删除当前行，P  粘贴)  
- `:wq！ +enter`  保存退出  
- `:q！+enter` 不保存退出 

##### 语法

` 1 2 3 4 5 /path/to/command arg1 arg2  `

- 第1个字段：分钟 (0-59) 
- 第2个字段：小时 (0-23) 
- 第3个字段：日期 (0-31) 
- 第4个字段：月份 (0-12 [12 代表 December])  
- 第5个字段：一周当中的某天 (0-7 [7 或 0 代表星期天]) 
- `/path/to/command` – 计划执行的脚本或命令的名称 

##### 操作符

- 星号 (`*`) : 指定每个可用的值。如在小时字段中，一个星号等同于每个小时；
- 逗号 (`,`) : 指定了多个值的列表，例如：1,5,10,15,20,25. 
- 横杠 (`-`) : 指定了值的范围，例如：5-15 ，等同于使用逗号操作符键入的  5,6,7,8,9,…,13,14,15。
- 分隔符 (`/`) : 此操作符指定了一个`步进值`，例如： 0-23/ (每小时被执行一 次，`默认步进为1`)。步进值也可以跟在星号操作符后边，如果你希望命令行每 2 小时执行一次，则可以使用 */2。

# openGauss辅助线程

- `GaussMaster`线程：openGauss的管理线程，也称为postmaster线程。用于数据库启停、消息转发等管理工作。
- `pagewriter`线程：负责将脏页数据从内存刷到磁盘中。
- `bgwriter`线程：负责将脏页数据拷贝至双写（double-writer）区域并落盘，然后将脏页转发给bgwriter子线程进行数据下盘操作。
- `walwriter`线程：负责将内存中的预写日志（WAL）页数据刷新到预写日志文件中，确保已提交的事务都被永久记录，不会丢失。
- `checkpoint`线程：周期性触发，每次触发会将全部脏页面刷到磁盘中。

# openEuler内置变量

1. **`$n` 表示 shell 程序（或过程）的第 n 个位置参数值，n=1..9**
    表示的是位置参数，第 `n` 个位置参数值，`n` 的取值范围是从 `1` 到 `9`。它用于获取传递给脚本的前 9 个参数。
2. **`$\*` 表示 shell 程序所有的位置参数组成的字符串**
   表示所有的位置参数（包括空格）的组合，作为一个单一的字符串。通常用来引用所有参数，但需要注意其与 `$@` 的区别。
3. **`$#` 表示 shell 程序的位置参数个数**
   表示传递给脚本的参数个数，准确来说就是位置参数的个数。

> `$0` 实际上表示脚本或程序的名称，而不是所有的参数。所有参数的变量是从 `$1` 开始，而 `$0` 是脚本或执行的命令名。
