# 操作系统experiment

1. `thread.join`

  1. 用于等待线程完成的函数。它的作用是<u>阻塞调用它的线程</u>，直到目标线程执行完毕。
  2. 只能<u>调用一次</u>：
      - 对同一个线程对象，`join()` 只能调用一次。如果再次调用，会导致程序崩溃。
  3. 不能与 `detach()` 同时使用：
      - 如果一个线程已经被 `detach()`，就不能再调用 `join()`，否则会导致未定义行为。
  4. **必须在线程对象销毁前调用**：
      - 如果线程对象在未调用 `join()` 或 `detach()` 的情况下销毁，会导致程序崩溃。

2. ### `std::unique_lock`

  1. C++ 标准库中提供的一种锁管理器，用于管理互斥锁（`std::mutex`）的所有权。
  2. 可以显式地锁定和解锁互斥锁（通过 `lock()` 和 `unlock()` 方法）。
  3. 支持延迟锁定（`defer_lock`）、尝试锁定（`try_to_lock`）和采用已有锁（`adopt_lock`）。
  4. 可以添加`{}`作用域来管理
  5. **自动管理锁的生命周期**：
    - 当 `std::unique_lock` 对象超出作用域时，会自动释放锁，避免死锁。
  6. **支持条件变量**：
    - `std::unique_lock` 是与 `std::condition_variable` 一起使用的推荐锁类型，因为它可以在等待时释放锁并在唤醒时重新获取锁。

3. **RAII**（**R**esource **A**cquisition **I**s **I**nitialization，资源获取即初始化）

  1. 一种 C++ 编程中的重要设计理念。
  2. 它的核心思想是将资源的获取和释放绑定到对象的生命周期中，通过构造函数获取资源，通过析构函数释放资源，从而确保资源的正确管理。
  3. 核心思想
    1. **资源获取**：在对象的构造函数中获取资源，例如内存分配、文件打开、锁定互斥锁等。
    2. **资源释放**：在对象的析构函数中释放资源，例如内存释放、文件关闭、解锁互斥锁等。
    3. **作用域管理**：资源的生命周期与对象的作用域绑定，当对象超出作用域时，资源会自动释放。
