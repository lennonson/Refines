## 自动装箱和拆箱

​	JDK5开始Java引入了自动装箱和拆箱机制，实现了包装类与基本类型之间的自动转换。

​		Java 为了`便于基本数据类型与 Object 类`之间的`转换处理`，提供有转向与拆箱的机制，所谓
的装箱指的是`将基本数据类型保存在包装类`之中，而所谓的拆箱指的就是`通过包装类获取基本数据类型对应的数值`。

> 包装类可以当成基本类型使用，前提是包装类的值不为null。如果包装类是null，自动拆箱会报空指针异常

## 享元模式

​		享元模式（Flyweight）是一种设计模式，其作用是通过共享对象来减少创建对象的数量，以达到节省内存提高性能的
目的。实现方式就是在第一次创建对象时将对象缓存起来，下次就不再创建，直接使用。

## UUID

- UUID是由一组32位数的16进制数字所构成，以连字号分隔的五组来显示，形式为8-4-4-4-12，总共有36个字符
  1. 优点：
     1. ·使用比较简单，自带的工具类生成即可。
     2. 性能好
  2. 缺点：
     1. ·由于主键生成的不规律性，一般主键都是自动创建索引，长度比较长，带来的问题是占用的存储空间比较大，查询效率不高。
     2. ·不适合作为分布式id。
