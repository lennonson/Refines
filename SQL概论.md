#概论 
1. 同型实体的集合称为实体集
2. 模式/==内模式==的映象可以保证数据和应用程序之间的==物理独立性==
3. 模式/==外模式==的映象可以保证数据和应用程序之间的==逻辑独立性==
4. 独立性
	- 数据库的==物理==独立性：应用程序与数据的物理存储相互独立
	- 数据库的==逻辑==独立性：应用程序与逻辑结构相互独立，逻辑结构改变，应用程序不用变
	- 数据库的==数据==独立性：应用程序和数据结构之间相互独立,互不影响。
5. 物理层是描述逻辑模型的物理实现，是数据库最底层的抽象。
6. 描述数据库中全体数据的全局逻辑结构和特征的是概念模式(模式/逻辑模式)。
7. 模式=概念模式=逻辑模式
8. E-R模型
	1. EER模型即(Enhanced Entity-Relatlonship)模型，扩展的实体-联系模型。
	2. E-R图模型的组成是由实体，属性和联系。
	3. ER模型中的联系是用来反映==实体内部==和==实体之间==的属性关系。
	4. ER模型与计算机无关，用户容易接受。
	5. 概念层模型是依客观需求对信息世界的建模，是现实世界到信息世界的==语义抽象==
	6. 但是ER图也有不足之处，它只说明了实体及实体间语义的联系，不能进一步说明详细的数据结构。
	7. E-R模型是一个==图示化模型==，E-R图提供了表示实体型、属性和联系的方法。==属性，用椭圆表示==，并用==无向边将其与相应的实体连接==起来。联系型本身也是一种实体型，也可以有属性。==实体型用矩形表示==，矩形框内写明实体名。。
	8. 一个1:n联系可以转换为一个独立的关系模式，也可以与联系的==n端实体==所对应的关系模式合并。
9. 关系模型用表格的数据来表示和实现实体间联系。

	13. 码
	    1. 当候选码多于一个时，选定其中一个作为主码
	    2. 主码可以是单个属性，也可以是属性组
	    3. 不包含在任何候选码中的属性叫做非主属性；
	    4. 单个属性是码；整个属性是码，称为全码。
	14. 多值依赖
	    1. 函数依赖可以看作多值依赖的特例。即若X$\rightarrow$Y，则X→→Y。
	    2. 分解规则对多值依赖不成立。
	    	1. 若函数依赖X→Y在R（U）上成立，则对于任何 Y'$~\subseteq~$Y 均有 X$~\rightarrow~$Y' 成立；
	    	2. 而多值依赖X→→Y若在（U）上成立，不能断言对于任何 Y'$~\subseteq~$Y 有 X→→Y' 成立。
	15. Armstrong公理系统
	    1. A1.自反律(Reflexivity）：$若~~~Y\subseteq X\subseteq U，则~~~X \rightarrow Y~~~成立。$
	    2. A2.增广律（Augmentation）：$~~~若Z\subseteq U且X→Y，则XZ→YZ成立。$
	    3. A3.传递律(Transitivity）：$若X→Y且Y→Z，则X→Z成立$
	    4. (A2，A3)  合并规则:        $由X→Y，X→Z,有X→YZ$    
	    5. (A2，A3)  伪传递规则： $由X→Y，WY→Z，有有XW→Z$。
	    6. (AI， A3)  分解规则：     $由X→Y及 Z \subseteq Y， 有X→Z$。
	    7. $由X→YZ，则X→Y，X→Z$
	       - 这是函数依赖的分解律（投影律）
	       - 如果X能决定YZ的组合，那么X当然能分别决定Y和Z
	16. 范式
	    1. 1NF：属性不可再分
	    2. 2NF：在1NF基础上消除了非主属性对码的部分函数依赖
	    3. 3NF：在2NF基础上消除了非主属性对码的传递函数依赖
	    4. BCNF：如果关系模式R $\subseteq$ 1NF、且所有的函数依赖X$\rightarrow$Y，决定因素X都包含了R的一个候选键，则称R属于BC范式。
	    5. 4NF：在3NF基础上消除了码间的部分和传递函数依赖
	    6. 补充
	    	1. 在函数依赖的范畴内，BCNF达到了最高的规范化程度
	    	2. 若只要求分解具有无损连接性，则模式分解一定能达到BCNF
	    	3. 模式分解中，既具有无损连接性，又保持函数依赖，则模式分解一定能达到3NF，但不一定能达到BCNF。
	    	4. 第四范式的讨论范围不仅涉及函数依赖，还涉及多值依赖
	17. 参照关系是从关系，被参考关系是主关系
	18. 以外码作为主码的关系称为被参考关系
	19. 关系操作中，五种基本操作是：并、差、笛卡尔积、投影和选择
	20. 如果两个关系没有公共属性，那么他们的自然连接就变为笛卡尔积
	21. 投影操作是从列的角度进行的操作
	22. 选择操作是从行的角度进行的操作
	23. 投影和选择都是一元关系操作符
	24. 连接操作是同时从行和列进行操作的
	25. 自然连接是构造新关系的有效方法，自然运算满足交换律
	26. 投影运算对并运算具有分配律，对交运算和差运算和除运算不具有分配律，==投影运算不具有交换律==
	27. 选择运算对并运算和交运算和差运算都具有分配律，选择运算满足交换律
	28. 选择运算对除运算不具有分配律
	29. 广义投影
	    1. 对关系R的广义投影操作，是在若干算术表达式上的投影，这些算术表达式只涉及常量和R中的属性
	    2. 广义投影是一种扩充的关系操作
	30. 完整性约束
	    1. 用户定义完整性约束主要是对属性的取值进行限定
	    2. 域完整性约束是一种用户定义完整性约束
	    3. 函数依赖和多值依赖属于域完整性约束中属性间的依赖关系
	    4. ==域完整性==约束就是指对关系中属性的取值的正确性进行限制，包括数据类型，精度、取值范围和是否允许空值等
	    5. ==域完整性==约束的取值范围可为==静态定义==和==动态定义==两种
	    6. ==域完整性==约束也包括定义属性间的依赖关系，包括函数依赖和多值依赖
	    7. 当执行插入数据操作时，首先检查实体完整性约束，再检查参照完整性约束最后检查用户定义完整性约束
	31. 一般来说，每个关系==至少有一个默认的超码==
	32. 一个关系的候选码是这个关系的==最小超码==
	33. 关系的==最大超码==是该关系的所有属性的集合
	34. ==超码==（Super Key）**定义**：能够唯一标识关系中每个元组的属性集合
	35. 笛卡尔积可表示为一个二维表，表中的每行对应于一个元组，每列对应于一个域

18. 分布式数据库
	1. 设计主要目标：数据处理的本地化，即将关系中的数据分成若干互不重叠的子集
	2. 垂直分片
		1. 将一个关系以列为单位“垂直地”分割
		2. 只保留关系地某些属性
		3. 每一个垂直分片都包含该关系地主键
	3. 完备性原则
		1. 指必须把全局关系的所有数据映射到片段中，绝不允许有属于全局关系的数据却不属于他的任何一个片段
	4. 分布式系统的所有问题是内部的、实现级别的问题，而不是外部的。
	5. 半连接操作是分布式数据库中一种有效的查询策略
